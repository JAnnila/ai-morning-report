## Repot ja yhteisön tekniikat (mitä kokeilla)

**Strix (usestrix/strix) – Autonomiset AI-penetraatiotestausagentit, 36 000+ tähteä, MIT, heinäkuun GitHub trending (aktiivinen kehitys heinäkuu 2026)**
Strix on avoimen lähdekoodin työkalu joka käyttää autonomisia moniagentteja haavoittuvuuksien löytämiseen, validointiin ja korjaukseen sovelluksissa – toimii kuten oikea hyökkääjä: ajaa koodia dynaamisesti, rakentaa PoC-exploitin eikä palauta vääriä positiivisia. Komponentit: HTTP Interception Proxy, selaimen hyökkäystestaus (XSS, CSRF, clickjacking, auth bypass), Shell & Command Execution, Custom Exploit Runtime, Rekon & OSINT. MIT-lisenssi; toimii omalla LLM API -avaimella (tuki Anthropic, OpenAI, Gemini). Saavuttanut heinäkuussa 2026 GitHub trending -sijoituksen merkittävällä uudella huomiomäärällä: tähtiä 36 000+ ja aktiivinen PR-kehitys. Alunperin julkaistu marraskuussa 2025, noussut nyt uudelleen laajaan yhteisöhuomioon. [GitHub: usestrix/strix](https://github.com/usestrix/strix) | [Help Net Security: Strix](https://www.helpnetsecurity.com/2025/11/17/strix-open-source-ai-agents-penetration-testing/) | [Strix.ai docs](https://www.strix.ai/)
*Miksi tärkeä / mitä kokeilla:* Erityisen ajankohtainen nyt kun Flowise CVE-2026-41264 (ks. Tietoturvavahti) muistuttaa että AI-rakentajatyökalut ovat alttiita prompt injection → RCE -ketjulle. Strix automatisoi juuri tällaisten haavoittuvuuksien löytämisen. Asenna paikallisesti (`npm install -g strix`), konfiguroi oma Anthropic API -avain, ja aja omaa Flowise-instanssiasi vasten ennen tuotantoon vientiä.

## Tietoturvavahti

**Microsoft Patch Tuesday heinäkuu 2026 (14.7.2026): 570 CVEä (ennätys) – VS Code 1.128.1 korjaa GitHub Copilot -tunnistetietovuodon (CVE-2026-47282) ja RCE:n (CVE-2026-50520)**
Microsoft julkaisi heinäkuun Patch Tuesdayn 14.7. – suurin päivityspaketti yrityksen historiassa: 570 haavoittuvuutta Windows-, Office-, Azure- ja kehittäjätyökalutuotteissa. AI-kehittäjille kriittiset korjaukset:

- **CVE-2026-47282 (tietopaljastavuus, tärkeä):** VS Code ja GitHub Copilot käsittelevät tunnistetietoja riittämättömästi suojattuna. Autentikoimaton hyökkääjä voi verkosta käsin hankkia Copilot-käyttämät GitHub-autentikointitokenit ja muut salaisuudet – suora riski yksityisiin repositorioihin ja CI/CD-pipelineihin. **Korjaus: VS Code 1.128.1**, julkaistu 14.7.2026.
- **CVE-2026-50520 (RCE, tärkeä):** Etäkoodinsuoritushaavoittuvuus Visual Studio Codessa.
- **CVE-2026-41109 (tietoturvaominaisuuden ohitus, tärkeä):** Vaikuttaa sekä VS Codeen että GitHub Copilotiin.

Qualys vahvistaa Patch Tuesday -katsauksessaan kaikki kolme AI-kehittäjätyökaluja koskevaa CVEä. Microsoftin mukaan Copilot-avustettu haavoittuvuuksien löytäminen selittää osaltaan CVE-volyymin kasvua. [VS Code 1.128.1 – Credential Leak Fix (Windows News)](https://windowsnews.ai/article/vs-code-11281-fixes-credential-leak-in-github-copilotupdate-now.438181) | [CVE-2026-47282 (Windows Forum)](https://windowsforum.com/threads/cve-2026-47282-update-vs-code-to-1-128-1-to-protect-copilot-credentials.438181/) | [Qualys: Patch Tuesday July 2026](https://blog.qualys.com/vulnerabilities-threat-research/2026/07/14/microsoft-and-adobe-patch-tuesday-july-2026-security-update-review) | [CyberSecurityNews: 570 CVEs](https://cybersecuritynews.com/microsoft-patch-tuesday-update-july-2026/)
*Miksi tärkeä / mitä kokeilla:* Päivitä VS Code **heti**: `Help → Check for Updates` tai lataa 1.128.1 suoraan. Enterprise-ympäristöissä: varmista pakollinen päivitys ennen huomista. CVE-2026-47282 on erityisen vakava CI/CD-putkissa jotka käyttävät GitHub Copilotia: vuotanut token antaa pääsyn kaikkiin yksityisiin repositorioihin joihin Copilotilla on lupa.

**Flowise CVE-2026-41264 (ZDI-26-364, julkaistu 14.7.2026): CVSS 9.8, autentikoimaton RCE CSV Agent -prompt injection kautta – Metasploit-moduuli saatavilla**
Trend Micron Zero Day Initiative julkaisi 14.7. advisory ZDI-26-364 (CVE-2026-41264): kriittinen haavoittuvuus FlowiseAI Flowise -versioissa 1.3.0–3.0.13. Haavoittuva polku: `CSV_Agents`-luokan `run`-metodi ajaa LLM:n generoimaa Python-koodia ilman asianmukaista sandboxia tai syötteen puhdistusta – hyökkääjä lataa haitallisen `.csv`-tiedoston, ohjaa LLM:n generoimaan koodia joka ajaa mielivaltaisia komentoja palvelimella. Ei vaadi autentikointia. Rapid7 julkaisi Metasploit-moduulin viikolla 11.7., mikä tekee exploitista automatisoitavissa olevan. Korjaus: Flowise 3.1.0. Yhteys aiempiin raportteihin: sama arkkitehtuuriongelma kuin PraisonAI CVE-2026-61447 (13.7. raportti) – LLM-generoidun koodin suorittaminen ilman eristystä. [GitHub Advisory: GHSA-3hjv-c53m-58jj](https://github.com/advisories/GHSA-3hjv-c53m-58jj) | [ZDI-26-364](https://www.zerodayinitiative.com/advisories/ZDI-26-364/) | [Rapid7: Metasploit Update](https://www.rapid7.com/blog/post/pt-weekly-metasploit-update-exploits-for-flowiseai-csv-agent-and-macos-package-kit/) | [GTK Cyber: CVE-2026-41264](https://gtkcyber.com/cve/CVE-2026-41264/)
*Miksi tärkeä / mitä kokeilla:* Tarkista Flowise-versio: `pip show flowise | grep Version` (tai tarkista Docker-imagen tagi). Jos versio on välillä 1.3.0–3.0.13, päivitä välittömästi 3.1.0+:aan. Laajempi malli: kolme kuukautta, kaksi eri avoimen lähdekoodin AI-rakentajatyökalua (PraisonAI ja Flowise), molemmat kriittinen RCE sama mekanismi. Auditoi kaikki no-code/low-code AI -frameworkit jotka suorittavat LLM-generoitua koodia: onko `exec()` tai `eval()` kutsuketjussa ilman sandboxia?

Anthropic lanseerasi 14.7. Claude for Teachersin: vahvistetuille US K-12 -opettajille ilmainen pääsy premium Claude -ominaisuuksiin, Claude Codeen, Coworkiin ja Education-spesifiseen Learning Commons -liittimeen joka kartoittaa Claude-tuotokset kaikkien 50 osavaltion akateemisiin standardeihin ja niiden aliosaamisiin. [Anthropic: Claude for Teachers](https://www.anthropic.com/news/claude-for-teachers) | [Chalkbeat 14.7.](https://www.chalkbeat.org/2026/07/14/anthropic-launches-claude-for-teachers-as-ai-companies-battle-for-classrooms/) | [9to5Mac 14.7.](https://9to5mac.com/2026/07/14/anthropic-is-giving-teachers-free-access-to-premium-claude-features-details-here/)
